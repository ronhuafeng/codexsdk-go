# v0.5 Migration

v0.5.0 makes the exact lifecycle partial-evidence contract consistent across
Start and Resume operations.

A successfully decoded lifecycle response remains observable as partial
evidence even when a required thread or turn identity is missing. Required
identities decide whether the SDK may continue execution; they do not decide
whether other decoded generated facts exist.

- Missing thread identity terminates with `ErrMissingThreadID` before
  `turn/start` is sent.
- Missing turn identity terminates with the new additive `ErrMissingTurnID`
  before a live turn stream is registered.
- Simple operations return the partial result and error together.
- Streaming operations return a non-nil terminal stream; `Wait`, `Result`, and
  `Err` expose immutable snapshots and the stable terminal cause.
- Failures before a lifecycle response decodes still return zero evidence or a
  nil stream with the direct error.

Use `errors.Is` rather than matching error text:

```go
result, err := runner.Resume(ctx, request)
if errors.Is(err, codexsdk.ErrMissingThreadID) {
	inspect(result.Resume)
}
if errors.Is(err, codexsdk.ErrMissingTurnID) {
	inspect(result.Run.Turn)
}
```

Neither identity error closes the Client. Applications may inspect the partial
facts and make a later valid call on the same Client.
