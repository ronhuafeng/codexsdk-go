# v0.2 Migration

## Lifecycle

Replace projected `ThreadClient` requests with exact generated params:

```go
result, err := client.ThreadRunner().Start(ctx, codexsdk.StartThreadRunRequest{
	Thread: protocolv2.ThreadStartParams{
		Model:     protocolv2.Value(model),
		Ephemeral: protocolv2.Value(true),
	},
	Turn: protocolv2.TurnStartParams{
		Input: []protocolv2.UserInput{
			protocolv2.NewUserInputText(protocolv2.UserInputText{Text: prompt}),
		},
	},
})
```

Use `result.Start` for the exact effective thread configuration and
`result.Run` for the terminal turn, last usage state, ordered notifications,
final-response projection, input statistics, and sanitized diagnostic refs.
When `err != nil`, inspect the returned result because it may contain a valid
start response and partial run evidence.

`Turn.ThreadID` is composition-owned and must be empty. Forking remains
available through the generated `client.Threads().Fork` method.

## Streaming

Replace `ThreadStream.Event` projections with `Stream.Notification`, which
returns the exact generated `protocolv2.ServerNotification`. `Stream.Result`
returns immutable snapshots during and after failures.

## Initialization And Handlers

Set `ClientOptions.Initialize` with exact `protocolv2.InitializeParams` instead
of `ClientName`, `ClientTitle`, or copied capability options.

`ServerNotificationHandler` receives every validated generated notification on
a bounded serial dispatcher. `ServerRequestHandler` receives the generated
request union and must return the matching typed constructor such as
`CommandExecutionApprovalResponse`. A mismatched or empty response fails
closed.

Leaving `ServerRequestHandler` nil no longer routes exact-run requests through
the deprecated projected callback or pending queue. The SDK responds
immediately with the generated denial/empty-answer shape when that is safe.
Requests that need application-owned data (authentication refresh, dynamic
tool output, or attestation) fail the run with `ErrExactServerRequest` after an
explicit JSON-RPC error. Inspect the returned result on failure for retained
partial notifications and run state.

The v0.1 projected lifecycle and callback declarations remain available but
deprecated until no earlier than v0.3.
