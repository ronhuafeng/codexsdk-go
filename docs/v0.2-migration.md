# v0.2 Migration

## Concrete root after v0.2.1

The post-v0.2.1 API replaces the SDK-owned `Client` interface with a concrete
`Client`; `New(ClientOptions)` now returns `*Client`. Code that relies on type
inference is unchanged. Explicit fields, parameters, or return values must move
from `codexsdk.Client` to `*codexsdk.Client`.

Do not replace the removed umbrella with another broad SDK interface. Define a
narrow consumer-owned interface around the operation used at that call site,
such as `ThreadRunner.Start` or `Models.List`. A zero-value `Client` is safe but
not connected: `Close` succeeds and operations return `ErrClientClosed`.

## Concrete generated facades in v0.4.0

The later concrete-facade change has its own
[v0.4 migration guide](v0.4-migration.md). It replaces generated facade
interfaces with exported concrete opaque values while keeping ordinary calls
unchanged.

## Lifecycle

Replace projected `ThreadClient` requests with exact generated params:

```go
result, err := client.ThreadRunner().Start(ctx, codexsdk.StartThreadRunRequest{
	Thread: protocolv2.ThreadStartParams{
		Model:     protocolv2.Value(model),
		Ephemeral: protocolv2.Value(true),
	},
	Turn: protocolv2.TurnStartParams{
		Input: []protocolv2.UserInput{
			protocolv2.NewUserInputText(protocolv2.UserInputText{Text: prompt}),
		},
	},
})
```

Use `result.Start` for the exact effective thread configuration and
`result.Run` for the terminal turn, last usage state, ordered notifications,
final-response projection, input statistics, and sanitized diagnostic refs.
When `err != nil`, inspect the returned result because it may contain a valid
start response and partial run evidence.

If a decoded `thread/start` response lacks its thread ID, `Start` and
`StartStream.Wait` preserve the decoded start facts and return
`ErrMissingThreadID`. No turn is started and the Client remains usable.

`Turn.ThreadID` is composition-owned and must be empty. Forking remains
available through the generated `client.Threads().Fork` method.

## Streaming

Replace `ThreadStream.Event` projections with `Stream.Notification`, which
returns the exact generated `protocolv2.ServerNotification`. `Stream.Result`
returns immutable snapshots during and after failures.

For non-destructive completion observation, call `Stream.Wait`. Concurrent
waiters receive independent immutable snapshots and the same stable terminal
cause. Canceling one wait context returns that caller's latest partial snapshot
and context error without canceling the shared run; `Stream.Close` remains the
explicit shared cancellation boundary. `Next(ctx)` retains its existing
single-consumer live-delivery behavior.

## Initialization And Handlers

Set `ClientOptions.Initialize` with exact `protocolv2.InitializeParams` instead
of `ClientName`, `ClientTitle`, or copied capability options.

`ServerNotificationHandler` receives every validated generated notification on
a bounded serial dispatcher. `ServerRequestHandler` receives the generated
request union and must return the matching typed constructor such as
`CommandExecutionApprovalResponse`. A mismatched or empty response fails
closed.

Leaving `ServerRequestHandler` nil no longer routes exact-run requests through
the deprecated projected callback or pending queue. The SDK responds
immediately with the generated denial/empty-answer shape when that is safe.
Requests that need application-owned data (authentication refresh, dynamic
tool output, or attestation) fail the run with `ErrExactServerRequest` after an
explicit JSON-RPC error. Inspect the returned result on failure for retained
partial notifications and run state.

## Removed v0.1 mapping

The v0.1 compatibility surface was removed in v0.3:

| Removed surface | Exact replacement |
| --- | --- |
| `ThreadClient` and `ThreadClientOptions` | `Client.ThreadRunner()` with exact start/resume request structs |
| `StartThreadRequest` and `ResumeThreadRequest` | `StartThreadRunRequest` and `ResumeThreadRunRequest` |
| `ForkThreadRequest` | `Client.Threads().Fork` with `protocolv2.ThreadForkParams` |
| `ThreadStream` and `ThreadEvent` | `Stream[R]` and `protocolv2.ServerNotification` |
| copied input/enums and `Text`, `TextAndFiles`, `Bool` | generated `protocolv2` values and constructors |
| projected results, items, usage, and warnings | generated response/notification facts in `ThreadRunResult` |
| `LegacyServerRequestHandler` and copied request/response models | `ServerRequestHandler` and typed response constructors |
| `ClientName`, `ClientTitle`, and copied capabilities | `ClientOptions.Initialize` |

There are no aliases or shims for removed upstream operations. Call the
corresponding generated facade when the checked-in protocol exposes one.
