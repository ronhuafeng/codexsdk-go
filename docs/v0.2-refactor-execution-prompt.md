# codexsdk-go v0.2 Refactor Execution Prompt

Implement the complete SDK refactor specified by
[v0.2-refactor-plan.md](v0.2-refactor-plan.md) in the current `codexsdk-go`
repository.

Read the entire plan before editing. It is authoritative for the handwritten
surface, generated boundary, lifecycle semantics, error behavior, migration,
and acceptance.

## Rules

- `codexsdk-go` only provides Go interfaces for Codex interaction.
- Generated `protocolv2`, method registry, and facade inventory are facts, not
  a handwritten redesign target. Change them only through their generators.
- Treat target declarations as proposed symbols to implement. Before using a
  generated type, verify it exists in the current generated source.
- Handwritten lifecycle APIs use exact generated thread/turn params and facts;
  do not create copied enums, reduced request models, or a second event union.
- Preserve partial start/resume/run results together with errors.
- Never execute user callbacks on the JSON-RPC read loop or silently drop a
  validated notification.
- Implement normal close and failure shutdown as separate state transitions.
  Record run evidence before callback enqueue. Do not add goroutine identity
  detection or `ErrReentrantClose`; handlers request shutdown by returning an
  error and must not call `Close`.
- Preserve unrelated user changes and untracked files.
- Do not add toolkit, adapter, provider-neutral LLM, workflow, or application
  policy dependencies.
- Keep published compatibility declarations only as documented legacy surface;
  do not let them shape the exact-param core.

## Execution Order

1. Recheck generated types, current exports, protocol coverage, and git state.
2. Implement and test exact lifecycle composition and result authority.
3. Implement exact streaming and full notification dispatch.
4. Complete server-request response coverage and public typed errors.
5. Add migration adapters, deprecations, docs, release metadata, and the
   handwritten export allowlist.
6. Run every repository generator/reproducibility check and completion audit.

## Validation

Run at minimum:

```sh
GOWORK=off go test ./...
GOWORK=off go vet ./...
```

Also run the repository's Python sync tests and reproducibility diffs for
method registry, protocol types, SDK surface, and capability metadata. Verify
the `contract:codexsdk-caller` block parses and every referenced generated type
exists. Compile a temporary downstream module against the actual SDK worktree.

Do not edit `llmcaller-codex-go`, change a remote tag, push, open a PR, or
publish a release without explicit user authorization. At the release gate,
report the exact tag needed by the caller and distinguish all-heads evidence
from released-version evidence.
