# v0.4 Migration

v0.4.0 changes each generated facade name from an SDK-owned interface to an
exported concrete opaque value. This is a documented pre-v1 breaking change.
Ordinary calls remain unchanged:

```go
result, err := client.Models().List(ctx, protocolv2.ModelListParams{})
```

Code that stored or implemented `codexsdk.Models` as an interface must instead
declare the narrow interface it consumes:

```go
type ModelLister interface {
	List(context.Context, protocolv2.ModelListParams) (protocolv2.ModelListResponse, error)
}
```

Both `client.Models()` and a consumer fake can satisfy `ModelLister`. Adding a
new generated method to the concrete `codexsdk.Models` value does not add an
implementation obligation to that fake. There is no compatibility alias for
the removed SDK-owned facade interface because the exported name now belongs
to the concrete value.

The root client remains `*codexsdk.Client`, and exact thread lifecycle remains
available through `client.ThreadRunner()`. See the
[pre-v1 public API boundary](public-api-boundary.md) for the complete ownership
model.
